# ShiDao MVP

Продуктовый MVP на Next.js с Supabase (Auth + Postgres).

## Языковая политика интерфейса
Текущий интерфейс **полностью на русском языке**: лэндинг, регистрация, вход, онбординг, кабинеты и подсказки.

## Запуск
```bash
npm install
npm run dev
```

## Основные маршруты
- `/` — обновлённый лэндинг с объяснением единого взрослого аккаунта и отдельного входа ученика.
- `/join` — единая регистрация для родителей и преподавателей (без выбора «родитель/преподаватель» в форме).
- `/login` — единая форма входа с 2 полями: email взрослого или логин ученика + пароль.
- `/onboarding` — первый вход взрослого с выбором ровно одного направления.
- `/dashboard/teacher` — зона преподавателя.
- `/dashboard/parent` — зона родителя.
- `/dashboard/student` — зона ученика.

## Как работает авторизация
- Источник истины — только **Supabase Auth**.
- Взрослые входят по email.
- Ученики входят по `login_identifier`; приложение сопоставляет логин с `student.auth_login_email` и выполняет вход через Supabase Auth.
- В приложении нет собственного хранения паролей и хешей.
- В интерфейсе используется только формулировка «пароль» (без обещаний отдельного PIN-флоу).

## Логика после входа
- Ученик сразу попадает в `/dashboard/student`.
- Взрослый без ролей попадает в `/onboarding`.
- Взрослый с ролями попадает в активную роль (`adult.current_role`) или в первую доступную роль.

## Онбординг взрослого (первый вход)
Ровно два варианта:
1. `Следить за обучением своего ребёнка`
2. `Преподавать Китайский язык`

В онбординге не предлагается опция «обе роли сразу».

## Переключение ролей
Для взрослых в шапке доступны:
- текущая роль: `Родитель` или `Преподаватель`;
- кнопка переключения на вторую роль.

Если второй роли ещё нет, она автоматически создаётся в `adult_role` и становится активной (`adult.current_role`).

## MVP-схема БД (Supabase)
Используются только таблицы домена:
- `adult`
- `adult_role`
- `school`
- `student`

Дополнительно для UX авторизации/ролей:
- `adult.current_role` — активная роль взрослого (`parent | teacher` или `NULL`);
- `adult_role.school_id` — контекст роли (`NULL` = personal, `NOT NULL` = school);
- `student.school_id` — контекст ученика (`NULL` = personal, `NOT NULL` = school);
- `student.login_identifier` — логин ученика для единой формы входа;
- `student.auth_login_email` — email ученика в Supabase Auth.

## Миграции
SQL-миграции находятся в `supabase/migrations`.
