# Авторизация и роли (MVP, март 2026)

## Ключевой принцип
Источник истины для аутентификации — только **Supabase Auth** (`auth.users`).

Доменные таблицы приложения не хранят пароли, хеши паролей или собственные credentials-таблицы.

## Доменная схема MVP
Используются только таблицы:
- `adult`
- `adult_role`
- `school`
- `student`

Контекст задаётся через nullable `school_id`:
- `school_id IS NULL` → personal-контекст;
- `school_id IS NOT NULL` → school-контекст.

## Единая регистрация взрослого
- Регистрация для взрослых выполняется через `/join`.
- Используется один канонический тип взрослого аккаунта.
- На регистрации нет выбора «родитель / преподаватель».
- На регистрации есть заметная подсказка, что ученики не регистрируются самостоятельно и должны входить по выданным данным.

После регистрации создаётся профиль `adult`, связанный с `auth.users` через `adult.auth_user_id`.

## Единая форма входа
Маршрут: `/login`.

Форма содержит ровно 2 поля:
1. `Email взрослого или логин ученика`
2. `Пароль`

Без вкладок и без предварительного выбора типа пользователя.

### Разрешение идентификатора
- Если введён email — это вход взрослого.
- Если введён не-email — трактуется как `student.login_identifier`; затем сопоставляется с `student.auth_login_email` и вход выполняется через Supabase Auth.

## Первый вход взрослого
Если у взрослого нет роли `parent`/`teacher`, выполняется переход на `/onboarding`.

Показываются только 2 варианта:
- `Следить за обучением своего ребёнка`
- `Преподавать Китайский язык`

На первом входе не предлагается опция «обе роли сразу».

## Переключение ролей взрослого
В шапке для взрослых показываются:
- текущая роль: `Родитель` или `Преподаватель`;
- действие переключения во вторую роль.

Поведение переключения:
- если роль уже есть в `adult_role` — активируется и выполняется редирект;
- если роли нет — запись создаётся, затем роль активируется.

Активная роль хранится в `adult.current_role`.

## Маршрутизация после входа
- `student` → `/dashboard/student`.
- `adult` без ролей → `/onboarding`.
- `adult` с ролями → на `adult.current_role` (или на первую доступную роль).

Студент не проходит взрослый онбординг.
