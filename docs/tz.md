# ТЗ (актуализация MVP-модели и UX авторизации)

Дата актуализации: **27 марта 2026**.

## 1) Язык интерфейса
Текущая продуктовая версия интерфейса — **только на русском языке**.

## 2) Целевая схема MVP
Используются только таблицы:
- `adult`
- `adult_role`
- `organization`
- `student`

## 3) Аутентификация
1. Вход и хранение паролей выполняются только через Supabase Auth.
2. Нельзя вводить собственное хранение паролей/хешей в публичных таблицах.
3. Не используются таблицы `student_credentials` и `student_guardian_link`.

## 4) Регистрация взрослого
1. Одна каноническая форма регистрации (`/auth/sign-up`).
2. Нет развилки parent vs teacher при регистрации.
3. Родитель и преподаватель регистрируются одинаково как взрослый аккаунт.

## 5) Единая форма входа
1. Одна форма входа (`/auth/sign-in`).
2. Ровно 2 поля:
   - идентификатор
   - пароль/PIN
3. Нет вкладок, переключателей и выбора типа пользователя до ввода данных.

## 6) Первый вход взрослого
Показывается только если у взрослого ещё нет роли.

Ровно два варианта выбора:
- `Следить за обучением своего ребёнка`
- `Преподавать Китайский язык`

Опция «обе роли одновременно» на этом шаге не используется.

## 7) Переключение ролей позже
В шапке взрослый пользователь может переключаться между ролями:
- `Родитель`
- `Преподаватель`

Если второй роли ещё нет, она создаётся в `adult_role` во время переключения.

## 8) Логика маршрутизации
- ученик → сразу в ученическую зону;
- взрослый без роли → в онбординг первого входа;
- взрослый с ролью(ями) → в текущую/последнюю активную роль.

## 9) Поля, добавленные для UX
- `adult.current_role` (`parent | teacher | NULL`) — активная роль взрослого.
- `student.login_identifier` — логин ученика для unified sign-in.
- `student.auth_login_email` — email пользователя ученика в Supabase Auth.
